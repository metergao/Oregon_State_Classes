Introduction



Pushbullet was founded in 2013 by Ryan Oldenburg, Andre von Houck, and Chris Hesse. Originally started as a side-project, Pushbullet bridges the gap between your phone, tablet, and computer, enabling them to work better together. From seeing your phone's notifications on your computer, to easily transferring links, files, and more between devices, Pushbullet saves you time by making what used to be difficult or impossible, easy. Headquartered in San Francisco, Pushbullet was founded to make all of your devices work better for you, no matter which one you happen to be using at the time.

Authentication


Access Token

The Pushbullet API handles authentication for each request with an "access token". Each access token is unique to exactly one Pushbullet account. This access token should be handled with care since anyone with this token can send and receive pushes. Pushbullet is also able to authenticate users with an OAuth2 request. When OAuhth is used it will have the user authenticated form the Pushbullet website. For this howto I will be showing each code example using an Access token. Get your access token form Account settings . You can test if you access token is valid by executing this:

curl --header ‘Access-Token: <ACCESS TOKEN>’ \
  https://api.pushbullet.com/v2/users/me

It should return JSON with your information including, name, email, image, and when your account was created.

{
   "active": true,
   "iden": "udDBb",
   "created": 1385410169.392102,

   "email_normalized": "quinnsam1@gmail.com",
   "name": "Sam Quinn",
   "max_upload_size": 26214400
}

Once you have verifed your access token from here on out you will need to include your Pushbullet Access Token with all other code samples.

Devices


List Devices

You can attach an unlimited amount of devices to your Pushbullet account so it is often useful to send pushes to individual devices that you own. To get a list of the Pushsbullet devices that the current user owns you can execute this;

curl --header ‘Access-Token: <ACCESS TOKEN>’ \
  https://api.pushbullet.com/v2/devices


The iden value is an alphanumeric string that uniquely identifies the device. This will be used later to send pushes to this specific devices instead of to every device.

This is all we need to know how to do for the device portion of this tutorial. you might want to write your iden value down since we will be using it in later sections.

This returns an array of every device that has ever been connected to Pushbullet, even the devices that are no longer in use. Within the returned JSON each device will have an active Value either True or False. Only active devices will be able to receive pushes.

Pushes


Viewing Pushes

Pushing will require us to actually send more data than just the access token unlike the previous requests covered earlier in this tutorial. I will cover the required data that needs to be sent for each section below.

curl --header ‘Access-Token: <ACCESS TOKEN>’
  --data-urlencode active="true" \
  --data-urlencode modified_after="1447866000" \
  --get \
  https://api.pushbullet.com/v2/pushes


You will notice that the parameters active and modified_after are added to the POST request. If you do not add these parameters you will get every Push that this account has sent, this could be desired but often times not.


The request above will return the data object listed on the right. As you might notice that this object is an array of pushes. Each push will be nested with in the array.

Sending Pushes

To send a push you will need to take advantage of curl’s data-binary function, which creates a dictionary object and is sent with a POST request.

Each push will require the following in the dictionary sent to the API.

  • Device_iden or email (channel or client_iden is also accepted but not covered in this tutorial.)
  • type [note, link, file]
  • title
  • body
  • [URL,File]

curl --header 'Access-Token: <ACCESS TOKEN>' \
 --header 'Content-Type: application/json' \
 --data-binary '{"device_iden":"<DEVICE IDEN>","body": \
 "This is Sam Quinns Pushbullet tutorial, \
 sample Push.","title":"Testing Pushing","type":"note"}' \
 --request POST \
 https://api.pushbullet.com/v2/pushes


If this push was pulled using the API form the "Viewing Pushes" section above, it will return a JSON object structured like.

The Pushes will send to all of the specified devices in your command and display them locally. Once the Push has been dismissed it will change the active value to false. If a push is dismised on one device it will be flaged as "non-active" and also be dismissed.

Subscribing a Channel

To subcribe to a channel you must send a Pushbullet binary object again to the API. The binary object that you send must contain the channel_tag. Pushbullet will use this value to search the channel database, if channel_tag is not found an invalid request JSON object will be returned.

curl --header ‘Access-Token: <ACCESS TOKEN>’ \
  --header 'Content-Type: application/json' \
  --data-binary '{"channel_tag":"cyanogenmod-updates"}' \
  --request POST \
  https://api.pushbullet.com/v2/subscriptions

If the command subscribe command worked as expected the API will return a JSON object with more information on the channel.

Applying the API


Unix Applications

Now that you are familiar with basics of the Pushbullet API I would like to show you a few things that you can use this tool for. I have included a Bash script here that is named “push” and for simplicity I will use this script in these examples. This script is written in Bash, and uses basic Unix utilities that should work out of the box on most Unix machines. You can use this script to go through the tutorial as I have added all of the topics that I have covered excluding the option to add subscriptions to this script.

You can now use Pushbullet to notify you when a large download has finished like this;

wget lage_file; push -m “Your download is done”


If you add this to your crontab profile you can have a notification every mornig with a fortune.

push -m “$(fortune)”

One of the things that I have added to my computer is a Pushbullet Linux Pam.d authentication module. Now I have Pushbullet send me a notification when ever anyone has SSHed into my computer. With the reliability and speed that Pushbullet offers it is a first choice utility for this type of security notification.

The possibilities are endless, while I did not cover it in this particular tutorial you may even send files with slight modification to the script provided.
